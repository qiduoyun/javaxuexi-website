log4j使用入门教程 炒菜  发布于 2015-11-13 23:21:56

    log4j用于输出各种不同级别的日志信息。常用的级别有：调试日志（debug）、正常信息日志（info）、警告日志（warn）、错误日志（error）。

    我们的程序运行过程中，通常会出现不同级别的错误、警告信息。特别是对于错误的信息，这些信息对于修复程序的漏洞有着很大指导性作用，所以我们有必要将这些我们需要的信息写入到日志中。

    下面以一个简单的例子来说明log4j的使用。

1、创建工程，导入log4j-1.2.17.jar包，在本例子的源码中含有这个jar包。

2、在src目录下创建properties文件：log4j.properties

    

log4j.rootLogger=debug,info, log, error
      
    ###Console ###  
    log4j.appender.info = org.apache.log4j.ConsoleAppender  
    log4j.appender.info.Target = System.out  
    log4j.appender.info.layout = org.apache.log4j.PatternLayout  
    log4j.appender.info.layout.ConversionPattern =  %d{ABSOLUTE} [ %t ] [ %p ]:%L - %m%n  
      
    ### Log ###    
    log4j.appender.log = org.apache.log4j.DailyRollingFileAppender  
    log4j.appender.log.File = log/log.log  
    log4j.appender.log.Append = true  
    log4j.appender.log.Threshold = debug  
    log4j.appender.log.DatePattern='.'yyyy-MM-dd  
    log4j.appender.log.layout = org.apache.log4j.PatternLayout  
    log4j.appender.log.layout.ConversionPattern = %-d{yyyy-MM-dd HH:mm:ss} [ %t ] %m%n  
      
    ### Error ###  
    log4j.appender.error = org.apache.log4j.DailyRollingFileAppender  
    log4j.appender.error.File = log/errorlog.log  
    log4j.appender.error.Append = true  
    log4j.appender.error.Threshold = ERROR   
    log4j.appender.error.DatePattern='.'yyyy-MM-dd  
    log4j.appender.error.layout = org.apache.log4j.PatternLayout  
    log4j.appender.error.layout.ConversionPattern =%-d{yyyy-MM-dd HH:mm:ss} [ %t ] %m%n

 

这里简单介绍一下：

Target，表示输出的目的，System.out是控制台输出。

File，表示日志保存的文件。

Threshold，表示日志写入的最低级别，如果日志级别在设置的最低级别及以上级别，都会写入日志。

ConversionPattern，日志格式。

 

3、在com.javaxuexi.log包下创建类：Log_Demo 

package com.javaxuexi.log;

import org.apache.log4j.Logger;

public class Log_Demo {
private static Logger logger=Logger.getLogger(Log_Demo.class);//创建log对象
    public static void main(String[] args) {
        // TODO Auto-generated method stub
        logger.debug("debug log message...");//调试日志
        logger.info("info log message...");//信息日志
        logger.warn("warn log message...");//警告日志
        logger.error("error log message...");//错误日志
    }

}

 

4、运行结果：

刷新工程，会看到多了个log目录和两个日志文件，打开就可以看到写入的日志信息了。
